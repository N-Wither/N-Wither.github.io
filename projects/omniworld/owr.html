时代先驱！国内最早跟进 1.20 的大型整合包

Omniworld 永远追随 MC 更新换代的脚步，哪怕是有着一大堆的模组要跟进。

如果你发现了更早的，告诉我，然后我删掉这一段。

我回来了！经典模组回归 1.20

Aether 模组在一众经典模组中打赢了复活赛来到了 1.20，现在骗过无数萌新的荧石传送门现在可在 Omniworld R 中成功激活。此外 Aether 的纹理也得到了更新，以更好地契合现在的 Minecraft 纹理。

除了 Aether 之外，Ender IO 也同样在 1.20 复活，如果你在为机器的面不够用或者线缆管道太乱而烦恼，现在你可以在一个方块的空间内放下四种管道了！

免责声明：Ender IO 目前不是完全体，如果你发现此模组的内容不完整，本人不承担任何责任。

格雷科技！史上最简单的格雷整合包

Omniworld R 首次引进了 GregTech 模组，此模组一直以来因其复杂度而出名，但是在 Omniworld R 中，我为此模组几乎所有的材料都制作了 Mystical Agriculture 的种子和 Productive Bees 的蜜蜂，你不需要为材料制作而等待，把你的时间用在刀刃上！

其人之道！在 Forge 使用 Fabric 模组

通过 Sinytra Connector，身为 Forge 整合包的 Omniworld R 加载了两个知名的 Fabric 模组：Better Nether 和 BetterEnd！自从 BetterEnd Forge 停止在了 1.16，Forge 玩家再也不能欣赏到美丽神秘的末地景色，现在我们又能骄傲地挺起胸膛，并向 Fabric 玩家炫耀 Forge 的模组内容之丰富了。

AE 回归！而且变得更好用了

现在 AE2 重新回归了 Omniworld，而且它现在没有了频道限制，AE Additions 模组也添加了不限种类的存储磁盘，此外 AE 和 RS 的处理器和成型/破坏核心可以互相通用！无论你是哪一边，现在都能满足！

遥遥领先！自主研发的矿物统一机制

1.16.5 的 Omniworld 中，矿物统一基于 Emendatus Enigmatica，而这个模组只有 1.16 和 1.19 版本，完美地避开了 1.18 和 1.20，给 Omniworld 系列带来了严重的卡脖子问题。

在学习了更多 JavaScript 知识以及认识到必须自主研发之后，我发明了 Emendatus Enigmatica JS，使用 JS 实现了 Emendatus Enigmatica 的大部分功能，而且扩展性非常强！
